4. builder pattern 학습의 필요성에 대한 사고하는 힘 기르기
	하나의 정보가 다수의 데이터를 보유하고 있다고 가정
	가령 객체 생성 즉 데이터를 사용 가능하게 생성함을 의미
	하나의 객체 생성시에 모든 보유한 멤버 변수값을 반드시 다 초기화하는것은 의무가 아님
	상황에 따르 초기화 안 하는 변수들도 다수 존재 가능
	
	가령
	 10개 변수 보유한 DTO 클래스 있다 가정

	 어떤 때는 5개의 데이터(변수)로 하나의 DTO
	 어떤 때는 7개의 데이터(변수)로 하나의 DTO 생성시에 기존 7개의 데이터가 아닌 다른데이터로 생성
	 DTO 클래스 개발 및 객체 생성 문법 고려?
	 
	 질문 
	 	개발 방법 1 : 기본 생성자로 객체 생성 후에 setXxx() 호출

	 	개발 방법 3 : 객체생성시 선별해서 필요한 멤버변수 값만 초기화 할 수 있는 방법 고민

5. 배열
	1. 다량의 데이터를 하나의 변수로 관리하는 구조
		- 개별 데이터 구조는 어떻게 분리? > 고유한 index > 0부터 시작
		- 0부터 시작
			: 문자열의 각 글자 의미하는 index도 0부터 시작
	2. 자바에서의 제약조건
		- 동일한 타입에 한해서만 배열에 저장 및 활용
	3. 기본 타입과 참조 타입으로 배열 생성 및 활용
	4. 배열 타입은 참조 타입
	5. 배열 저장 영역 - heap 
	6. 문법
		1. [] 배열 의미하는 표현
			- 선언, 생성, 초기화 동시
				- 타입 [] 변수 = {값1, 값2, ...}
				- new 없이도 객체 생성(String과 배열만 해당)
				
			- 선언, 생성 초기화 분리해서 개발
				- 선언 : 타입 [] 변수명;
				- 생성 : 변수명 = new 타입[메모리수];
				- 초기화 : 변수명[index값] = 값;
		2. [] = 1차원 / [][] = 2차원 / [][][] = 3차원
	7. lenght
		- 배열이 메모리에 생성시 자동 생성되는 배열만의 변수
		- 생성된 배열의 메모리 개수
	
	 	 	
6. 리뷰
	데이터 타입 종류
		기본 - byte/short/int/long | float/double | char | boolean	
				값만 표기에 맞게 변수에 대입 활용
	 	참조 - 객체타입
	 		api제공하는 모든 클래스 타입/사용자가 개발하는 클래스 타입 / 배열
	 		객체 생성(new 생성자([..]))해서 변수에 대입해 활용
	 		객체가 보유한 변수와 메소드 호출로 활용
	 		
	 데이터 타입 사용 위치
	 	변수 타입(멤버 변수(heap), 로컬 변수(stack)) / 메소드 반환타입
